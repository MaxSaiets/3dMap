# Швидкий старт

## Крок 1: Встановлення Backend

```bash
cd H:\3dMap\backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

## Крок 2: Запуск Backend

```bash
python run.py
```

Backend запуститься на `http://localhost:8000`

## Крок 3: Встановлення Frontend

Відкрийте новий термінал:

```bash
cd H:\3dMap\frontend
npm install
```

## Крок 4: Налаштування Frontend

Створіть файл `frontend/.env.local`:

```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## Крок 5: Запуск Frontend

```bash
npm run dev
```

Frontend запуститься на `http://localhost:3000`

## Крок 6: Використання

1. Відкрийте браузер: `http://localhost:3000`
2. Виберіть область на карті (використовуйте інструменти малювання)
3. Налаштуйте параметри в бічній панелі
4. Натисніть "Згенерувати модель"
5. Дочекайтеся завершення (може зайняти 10-60 секунд)
6. Завантажте згенерований файл

## Тестова область

Для тестування виберіть невелику область (наприклад, 500x500 метрів) в центрі міста.

Рекомендовані координати для тесту (Київ):

- Північ: 50.455
- Південь: 50.450
- Схід: 30.530
- Захід: 30.520

## Усунення проблем

### Backend не запускається

Перевірте, що Python 3.9+ встановлено:

```bash
python --version
```

### Помилки встановлення залежностей

На Windows може знадобитися Visual C++ Build Tools для деяких пакетів.

Альтернатива - використовуйте conda:

```bash
conda install -c conda-forge osmnx geopandas trimesh
```

### Frontend не підключається до Backend

1. Перевірте, що backend запущений на порту 8000
2. Перевірте файл `.env.local`
3. Перевірте налаштування CORS в `backend/main.py`

### Помилка генерації моделі

- Перевірте, що область не занадто велика (максимум 2x2 км)
- Перевірте, що в області є будівлі та дороги
- Перегляньте логи backend для деталей помилки
