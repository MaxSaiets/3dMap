# Діагностика проблем генерації моделей

## Проблема: "Створено 0 будівель"

### Можливі причини:

1. **Область занадто мала або не містить будівель**
   - Перевірте, що обрана область містить будівлі в OSM
   - Спробуйте вибрати більшу область або інше місце

2. **Геометрія будівель невалідна**
   - Код автоматично намагається виправити через `buffer(0)`
   - Перевірте логи backend на помилки обробки

3. **Проблеми з проекцією координат**
   - Будівлі можуть не проектуватися правильно з WGS84 в метри
   - Перевірте логи на помилки проекції

### Рішення:

1. **Перевірте логи backend** - там будуть детальні повідомлення про помилки
2. **Спробуйте іншу область** - виберіть місце з більшою кількістю будівель
3. **Збільште область** - невеликі області можуть не містити будівель

## Проблема: Рельєф плоский

### Поточна реалізація:

За замовчуванням використовується **синтетичний рельєф** (хвильовий паттерн) для демонстрації.

### Для реального рельєфу:

1. **Безкоштовно (без API ключа)**:
   - OpenTopography - потрібно реалізувати завантаження тайлів
   - Локальні DEM файли - завантажте SRTM/NASADEM тайли

2. **З API ключем**:
   - Mapbox Terrain API - додайте `MAPBOX_API_KEY` в `.env`
   - Google Elevation API - додайте `GOOGLE_ELEVATION_API_KEY` в `.env`

### Як додати API ключ:

1. Створіть файл `backend/.env`:
   ```
   MAPBOX_API_KEY=your_key_here
   ```

2. Або встановіть змінну середовища:
   ```bash
   export MAPBOX_API_KEY=your_key_here
   ```

## Проблема: Дороги не відображаються

### Можливі причини:

1. **Немає доріг в обраній області**
2. **Проблеми з буферизацією доріг**
3. **Помилки об'єднання доріг**

### Рішення:

Перевірте логи backend - там будуть повідомлення про обробку доріг.

## Детальне логування

Тепер код виводить детальну інформацію:
- Кількість завантажених будівель/доріг
- Кількість оброблених об'єктів
- Помилки обробки з traceback
- Статус експорту

Перевірте консоль backend для деталей.
